# 合約基底架構（合約領域・系統脈絡知識草稿）

> 用途：作為「合約」領域的**系統脈絡（基底架構知識）**，每輪固定注入，讓 AI 客服能**判斷**（解讀狀態、推斷下一步、依規則回答），而非只複述 API 回傳的數字。
> 性質：領域框架（穩定），與 API 的「即時現值」互補。事實依據見 research.md（jgb2 真實程式）。
> 撰寫原則：先述情境再帶條件、不模板化（知識品質標準）。

## 一、合約狀態怎麼看：`status` 與 `bit_status` 是兩件事

一份合約有兩個狀態欄位，意義不同，別混用：
- **`status`（目前階段）**：合約此刻所在的**單一**階段。回答「現在到哪一步」看這個。
- **`bit_status`（歷程位元）**：**累積**記錄合約走過的每個里程碑（是一個位元遮罩，可能同時含多個里程碑）。回答「已完成哪些步驟」時參考這個，須逐位元拆解，不能當單一數字查表。

兩者用同一組 12 個里程碑值（下表）。

## 二、12 個里程碑與 4 大階段

合約生命週期分四大階段：**簽約前 → 執行中 → 提前解約（執行中的岔路）→ 歷史**。

| 值 | 里程碑 | 階段 | 白話 |
|---|---|---|---|
| 1 | 合約建立 | 簽約前 | 剛建好，還沒送出邀請 |
| 2 | 已送簽約邀請 | 簽約前 | 等**租客**簽回 |
| 4 | 租客已簽 | 簽約前 | 等**房東**簽名 |
| 8 | 雙方簽名完成 | 執行中 | 合約正式生效 |
| 16 | 點交送出 | 執行中 | 已發點交（交屋）清單，等租客同意 |
| 32 | 點交完成 | 執行中 | 租客已同意點交（已交屋）|
| 64 | 點退送出 | 執行中 | 已發點退（退租）清單，等租客同意 |
| 128 | 點退完成 | 執行中 | 租客已同意點退（已退租）|
| 256 | 提前解約中 | 執行中 | 提前解約申請進行中 |
| 512 | 提前解約已確認 | 執行中 | 提前解約確認 |
| 1024 | 歷史合約 | 歷史 | 合約結束 |
| 2048 | 歷史(生命週期結束) | 歷史 | 完全結束 |

> 注意用詞：**點交＝交屋（move-in）**、**點退＝退租（move-out）**，別搞反。

## 三、每個階段「接下來會怎樣 / 現在能做什麼」

這是判斷的重點——使用者問「我這份怎麼了/下一步」時，依現階段推斷：

- **待租客簽回（2）**：下一步等租客簽名。房東無法先動點交/續約。
- **待房東簽名（4）**：租客簽了，等房東簽。房東簽完即「雙方簽名完成」生效。
- **雙方簽名完成（8）**：合約生效。此時可：**發點交**（需已連結租客）、**提前解約**（租客需合約允許提前解約；房東可）、**續約**（需已連結且未過期）。
- **點交完成（32）**：可**點退**、可**提前解約**。
- **點退（64/128）進行中或完成**：不可再點交；接近收尾。
- **提前解約中（256）**：不可點交；解約確認後 date_end 會被改為提前解約的到期日，之後轉歷史。
- **歷史（1024/2048）**：合約已結束，僅供查閱。

**可否動作的關鍵前提**（回答「為什麼不能…」時用）：
- 點交：須已連結租客 + 合約已生效（雙方簽名完成）；已在點交/點退/歷史則不可。
- 點退：須在租期開始日之後；狀態須為生效/點交完成/歷史。
- 提前解約：僅生效或點交完成可申請；租客須「合約允許提前解約」；續約進行中不可。

## 四、提前解約違約金怎麼算

僅當合約「允許提前解約」時適用。規則有**三種型態**，且分**兩階**（依是否在通知期限內）：

- **通知期限**：一方須於「提前解約通知天數」前通知他方。
- **型態 0（按月數）**：違約金 = 月租 × 月數。**通知期限內**用第一階月數、**超過**用第二階月數。
- **型態 1（固定金額）**：直接一筆固定金額（同樣分兩階）。
- **型態 2（其他）**：以合約後文補充條款的自訂文字為準（沒有標準公式）。

回答時：先確認該合約是否允許提前解約與型態，再帶對應算法；型態 2 一律導向補充條款/專人，不自行推算。

## 五、滯納金怎麼算

僅當該合約「啟用滯納金」時適用（此設定承自房東）：
- 逾期超過「緩衝天數」後，按「滯納金百分比 %」計收。
- 例：緩衝 7 天、5% → 到期後第 8 天起，以 5% 計滯納金。

## 六、續約與同物件多份合約（很重要）

- **續約是父子鏈**：續約合約會指向原合約（父約）。同一個物件、同一租客，隨著續約/不同租期，會有**多份合約**，而且**物件名稱（title）都一樣**。
- 付款資訊綁在**父約**。
- 查詢時只回「最新且有效」的當前列。
- **對客服的意義**：使用者只講物件名稱時，往往對應**多份**合約（不同期間/狀態）。此時要**帶期間與狀態列出候選**讓對方選，或反問更明確條件；不能假設只有一份。

## 七、關鍵欄位語義（回答時的詞彙表）

- **title**：物件/合約名稱（搜尋鍵；同物件多份會重複）。
- **date_start ~ date_end**：租期；提前解約會把 date_end 改為解約到期日。
- **rent / deposit_amount**：月租 / 押金。
- **法定用途**：1＝一般住宅、2＝社會住宅（社宅有專屬規則）。
- **allow_early_termination / early_termination_days**：是否允許提前解約 / 通知天數。
- **father_id**：續約父約參照（判斷是否為續約）。
- **is_history / active / is_newest**：是否歷史 / 是否有效 / 是否續約鏈當前列。
