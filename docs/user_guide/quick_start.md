# 知識完善迴圈系統 - 快速入門

## 5 分鐘快速上手

### 步驟 1：啟動迴圈 (1 分鐘)

1. 訪問 `http://localhost:8087/backtest`
2. 確認在「迴圈管理」Tab
3. 點擊「**啟動新迴圈**」按鈕
4. 填寫表單：
   ```
   迴圈名稱：測試批次
   業者：富喬物業管理顧問股份有限公司
   批次大小：50
   目標通過率：85%
   ```
5. 點擊「確認啟動」

**✅ 完成**：系統建立迴圈 #3，自動選取 50 題測試集

---

### 步驟 2：執行第一次迭代 (10-15 分鐘)

1. 在迴圈列表找到剛建立的迴圈
2. 點擊「**詳情**」按鈕
3. 點擊「**執行迭代**」按鈕
4. 確認執行（選擇非同步模式）
5. 等待系統自動執行：
   ```
   回測中...    (5-10 分鐘)
   分析中...    (1 分鐘)
   生成中...    (2-5 分鐘)
   審核中 ✓     (等待人工)
   ```

**✅ 完成**：系統生成 3-8 個待審核知識

---

### 步驟 3：審核知識 (5-10 分鐘)

1. 切換到「**知識審核**」Tab
2. 查看待審核知識清單
3. 檢查每個知識：
   - 問題是否正確？
   - 答案是否完整？
   - 是否有重複警告？
4. 選擇動作：
   - **批准**：知識立即生效
   - **修改**：編輯後批准
   - **拒絕**：不採用

**💡 快速批量審核**：
- 勾選多個知識
- 點擊「批量批准」
- 一次審核 10-50 個

**✅ 完成**：知識同步到正式庫，下次回測會使用新知識

---

### 步驟 4：查看改善效果 (2 分鐘)

1. 回到「**迴圈管理**」Tab
2. 點擊迴圈詳情
3. 點擊「執行迭代」（第 2 次）
4. 等待執行完成
5. 切換到「**回測結果**」Tab
6. 選擇對應的迴圈 ID
7. 查看迭代對比：
   ```
   第 1 次迭代：通過率 60% (30/50)
   第 2 次迭代：通過率 74% (37/50) ↑ 14% ✅
   ```

**✅ 完成**：確認知識改善有效

---

### 步驟 5：達成目標或繼續迭代

**如果通過率 < 85%**：
- 重複步驟 2-4
- 最多執行 10 次迭代

**如果通過率 ≥ 85%**：
1. 點擊「完成批次」
2. 可選：點擊「啟動下一批次」
3. 系統自動選取新的 50 題

## 常用操作

### 如何切換 Tab？

點擊頂部導航：
```
[🔄 迴圈管理] [📝 知識審核] [📊 回測結果] [✅ 驗證回測]
```

---

### 如何查看特定迴圈的回測結果？

**方法 1**：URL 參數
```
/backtest?tab=backtest-results&loopId=3
```

**方法 2**：手動選擇
1. 切換到「回測結果」Tab
2. 在「回測記錄」下拉選單選擇對應的 Run ID

---

### 如何批量審核知識？

1. 切換到「知識審核」Tab
2. 勾選多個知識（按住 Shift 可連續選取）
3. 點擊「批量批准」或「批量拒絕」
4. 等待進度完成

**💡 建議**：一次審核 10-20 個，避免審核疲勞

---

### 如何查看迭代改善趨勢？

1. 切換到「回測結果」Tab
2. URL 加上 `?loopId=3`
3. 查看「迭代對比」區塊：
   ```
   第 1 次：60%
   第 2 次：74% ↑ 14%
   第 3 次：82% ↑ 8%
   第 4 次：88% ↑ 6% ✅ 達標
   ```

---

### 如何暫停迴圈？

1. 迴圈詳情 Modal
2. 點擊「暫停」按鈕
3. 系統標記為 `PAUSED`
4. 後續可點擊「恢復」繼續執行

---

### 如何取消迴圈？

1. 迴圈詳情 Modal
2. 點擊「取消」按鈕
3. 系統標記為 `CANCELLED`
4. 無法恢復（請謹慎操作）

---

## 常見問題

### Q: 為什麼通過率沒有提升？
**A**: 可能原因：
1. 審核時拒絕了太多知識 → 重新審核並批准
2. 生成的知識品質不佳 → 修改後再批准
3. 測試題目難度太高 → 需要更多迭代

---

### Q: 可以同時執行多個迴圈嗎？
**A**: 可以，但建議：
- 一次專注於 1-2 個迴圈
- 避免審核時混淆不同批次的知識

---

### Q: 測試集可以修改嗎？
**A**: 不可以。固定測試集是為了：
- 確保迭代間的結果可比較
- 避免「換題目導致通過率變化」的誤判

如需測試不同題目，請啟動新迴圈。

---

### Q: 如何知道何時該完成批次？
**A**: 建議標準：
- 通過率 ≥ 85%（達成目標）
- 連續 2 次迭代改善 < 2%（邊際遞減）
- 已執行 10 次迭代（達到上限）

---

### Q: 驗證回測 Tab 為什麼顯示「開發中」？
**A**: 此功能為可選功能，目前尚未實作。

**替代方案**：
- 使用「執行迭代」功能進行完整回測
- 可達到相同的驗證效果
