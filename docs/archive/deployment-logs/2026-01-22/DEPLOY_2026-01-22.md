## 📌 本次更新概述

### 🎯 核心目標
解決"每次推版都會漏掉欄位"的問題，建立完整的資料庫 Migration 追蹤和執行機制。

### ✨ 主要功能

#### 1. Migration 追蹤系統
- ✅ 創建 `schema_migrations` 表，追蹤所有已執行的 migration
- ✅ 每個 migration 執行後自動記錄時間、狀態、執行時間
- ✅ 防止重複執行已完成的 migration

#### 2. 自動執行腳本
**檔案**: `database/run_migrations.sh`

**功能特性**:
- ✅ **Dry-run 模式**: 預覽即將執行的 migration，不實際執行
- ✅ **自動備份**: 執行前自動備份資料庫到 `database/backups/`
- ✅ **交互式確認**: 可選的手動確認機制
- ✅ **冪等性**: 已執行的 migration 自動跳過
- ✅ **錯誤處理**: 失敗時立即停止，保留錯誤日誌
- ✅ **回滾指南**: 失敗時自動顯示回滾命令

#### 3. 完整文檔
**檔案**: `database/migrations/README.md`

包含：
- 17 個 migration 的完整清單和說明
- 自動執行腳本使用指南
- 新增 migration 的標準流程
- 常見問題解答（FAQ）
- 回滾策略

### 步驟 3: Dry-run 預覽（重要！）

```bash
./database/run_migrations.sh docker-compose.prod.yml --dry-run
```

**可能的結果：**

#### 情況 A：資料庫結構已是最新（生產環境首次使用此系統）

```
✓ 所有 migration 都已執行，無需執行
   已執行: 17 個
```

**處理方式**：跳到步驟 5（只需重啟服務）

#### 情況 B：發現待執行的 migration

```
⚠️  發現 17 個待執行的 migration:
   - 000_create_schema_migrations
   - add_action_type_and_api_config
   ... (其他 15 個)
```

**處理方式**：繼續步驟 4

---

### 步驟 4: 執行 Migration（僅當步驟 3 發現待執行的 migration）

#### 4.1 手動備份（雙重保險）

```bash
# 腳本會自動備份，但額外手動備份更安全
docker-compose -f docker-compose.prod.yml exec -T postgres \
  pg_dump -U aichatbot aichatbot_admin > backup_prod_$(date +%Y%m%d_%H%M%S).sql

# 確認備份成功
ls -lh backup_prod_*.sql
```

#### 4.2 執行 Migration

**方案 A：自動執行（推薦）**
```bash
./database/run_migrations.sh docker-compose.prod.yml
```

腳本會：
1. ✅ 檢查資料庫服務
2. ✅ 掃描所有 migration 文件
3. ✅ 自動備份到 `database/backups/`
4. ✅ 按順序執行未執行的 migration
5. ✅ 記錄執行結果
6. ✅ 顯示執行歷史

**方案 B：交互式執行（更謹慎）**
```bash
./database/run_migrations.sh docker-compose.prod.yml --interactive
```

執行時會詢問確認：
```
即將執行 17 個 migration，是否繼續？(y/N)
```
輸入 `y` 才會實際執行。

#### 4.3 驗證執行結果

```bash
# 再次 dry-run，確認所有 migration 已執行
./database/run_migrations.sh docker-compose.prod.yml --dry-run
```

**預期輸出**：
```
✓ 所有 migration 都已執行，無需執行
   已執行: 17 個
```

## 🆘 故障排除

### 問題 1: Migration 執行失敗

**症狀**：腳本顯示 `✗ migration_name (失敗)`

**處理步驟**：
```bash
# 1. 查看錯誤日誌
ls -lt /tmp/migration_*.log | head -1
cat /tmp/migration_NAME_*.log

# 2. 使用自動備份回滾
docker exec -i aichatbot-postgres psql -U aichatbot aichatbot_admin < database/backups/backup_before_migration_*.sql

# 3. 或使用手動備份回滾
docker exec -i aichatbot-postgres psql -U aichatbot aichatbot_admin < backup_prod_*.sql

# 4. 檢查 schema_migrations 表
docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin -c "
    SELECT * FROM schema_migrations WHERE success = false;
  "

# 5. 修正問題後，從 schema_migrations 刪除失敗記錄
docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin -c "
    DELETE FROM schema_migrations WHERE migration_name = 'failed_migration_name';
  "

# 6. 重新執行
./database/run_migrations.sh docker-compose.prod.yml
```

---

### 問題 2: 腳本顯示所有 migration 已執行，但資料庫欄位缺失

**症狀**：Dry-run 顯示都已執行，但 API 報錯找不到欄位

**可能原因**：有人手動在 `schema_migrations` 插入了記錄，但沒有實際執行 SQL

**處理步驟**：
```bash
# 1. 檢查缺失的欄位
docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin -c "\d knowledge_base"

# 2. 如果確認欄位缺失，手動執行對應的 migration
cat database/migrations/add_action_type_and_api_config.sql | \
  docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin

# 3. 重啟服務
docker-compose -f docker-compose.prod.yml restart
```

---

### 問題 3: 服務啟動後日誌出現資料庫相關錯誤

**症狀**：`column "action_type" does not exist` 或類似錯誤

**處理步驟**：
```bash
# 1. 確認 migration 執行狀態
./database/run_migrations.sh docker-compose.prod.yml --dry-run

# 2. 查看 migration 執行歷史
docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin -c "
    SELECT migration_name, executed_at, success
    FROM schema_migrations
    ORDER BY executed_at DESC
    LIMIT 10;
  "

# 3. 檢查實際的資料表結構
docker-compose -f docker-compose.prod.yml exec -T postgres \
  psql -U aichatbot -d aichatbot_admin -c "\d knowledge_base"

# 4. 如果不一致，參考問題 2 的處理步驟
```

### 方案 A：只回滾程式碼（推薦，如果資料庫 migration 成功）

```bash
git log --oneline -5

docker-compose -f docker-compose.prod.yml restart
```

⚠️ **警告**：此操作會回滾資料庫變更，請確保已備份！

```bash
# 1. 使用備份恢復資料庫
docker exec -i aichatbot-postgres psql -U aichatbot aichatbot_admin < backup_prod_*.sql

# 2. 回滾程式碼
git checkout [previous_commit_hash]

# 3. 重新構建並啟動
docker-compose -f docker-compose.prod.yml down
docker-compose -f docker-compose.prod.yml build --no-cache
docker-compose -f docker-compose.prod.yml up -d
```
