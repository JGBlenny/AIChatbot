# 前端功能缺口清單

根據 requirements.md 規範，以下是目前**缺少但應該實作**的前端功能：

---

## 1. Loop 管理界面（Critical）

### 功能需求

**路徑建議**：`/loop-management` 或 `/knowledge-loops`

**主要功能**：

#### A. Loop 列表視圖
```
+------------------------------------------------------------------+
| 知識完善迴圈管理                                [+ 啟動新迴圈] |
+------------------------------------------------------------------+
| Loop ID | 名稱            | 狀態      | 迭代  | 通過率 | 操作    |
|---------|-----------------|-----------|-------|--------|---------|
| #15     | 500題測試       | RUNNING   | 2/10  | 72%    | [繼續]  |
| #14     | 50題驗證-重測   | COMPLETED | 3/10  | 88%    | [查看]  |
| #13     | 50題驗證-初次   | COMPLETED | 3/10  | 86%    | [查看]  |
| #12     | 測試迴圈        | FAILED    | 5/10  | 65%    | [重試]  |
+------------------------------------------------------------------+
```

**顯示欄位**：
- Loop ID
- Loop 名稱
- 狀態（RUNNING / REVIEWING / COMPLETED / PAUSED）
- 當前迭代 / 最大迭代
- 當前通過率 / 目標通過率
- 測試題數（scenario_ids 陣列長度）
- 建立時間
- 操作按鈕（繼續迭代 / 暫停 / 取消 / 查看詳情）

**API 需求**：
```
GET /api/v1/loops?page=1&page_size=20
```

---

#### B. 啟動新迴圈（對話框）

**觸發方式**：點擊「+ 啟動新迴圈」按鈕

**表單欄位**：
```
+-----------------------------------------------+
| 啟動新的知識完善迴圈                           |
+-----------------------------------------------+
| 迴圈名稱：                                     |
| [_______________]  (例如：500題測試)           |
|                                               |
| 業者：                                         |
| [下拉選單：業者 1, 業者 2, ...]                |
|                                               |
| 測試規模：                                     |
| [ ] 50 題（快速驗證）                          |
| [x] 500 題（完整測試）                         |
| [ ] 3000 題（全面評估）                        |
| [ ] 自訂：[____] 題                            |
|                                               |
| 測試情境選取策略：                             |
| [x] 分層隨機抽樣（推薦）                       |
| [ ] 完全隨機                                   |
| [ ] 順序選取                                   |
|                                               |
| 目標通過率：                                   |
| [____85____]% （預設 85%）                    |
|                                               |
| 最大迭代次數：                                 |
| [____10____]  （預設 10 次）                  |
|                                               |
| 回應類型判斷模式：                             |
| [x] AI 輔助判斷（推薦）                        |
| [ ] 規則判斷                                   |
|                                               |
|                [取消]  [啟動迴圈]              |
+-----------------------------------------------+
```

**API 需求**：
```
POST /api/v1/loops/start
Request Body:
{
  "vendor_id": 1,
  "loop_name": "500題測試",
  "batch_size": 500,
  "target_pass_rate": 0.85,
  "max_iterations": 10,
  "selection_strategy": "stratified_random",
  "action_type_mode": "ai_assisted"
}

Response:
{
  "loop_id": 15,
  "status": "RUNNING",
  "message": "迴圈已啟動，正在執行第一次迭代..."
}
```

#### C. Loop 詳細視圖

**路徑**：`/loop-management/{loop_id}`

**顯示內容**：

**1. 基本資訊卡片**
```
+----------------------------------------------------------+
| Loop #15 - 500題測試                                     |
+----------------------------------------------------------+
| 狀態：🟢 RUNNING (審核中)                                |
| 目前迭代：2 / 10                                         |
| 目標通過率：85% (目前：72%)                              |
| 測試規模：500 題（scenario_ids 已固定）                  |
| 建立時間：2026-03-27 14:30                               |
| 最後更新：2026-03-27 16:15                               |
+----------------------------------------------------------+
```

**2. 迭代歷史表格**
```
+------------------------------------------------------------------+
| 迭代 | 回測結果      | 知識生成    | 成本      | 改善幅度 | 狀態   |
|------|--------------|------------|-----------|---------|--------|
| 2    | 360/500 (72%)| SOP:5, KB:8| $2.34     | +12%    | 審核中 |
| 1    | 300/500 (60%)| SOP:6, KB:7| $2.10     | -       | 已完成 |
+------------------------------------------------------------------+
```

**3. 快速操作按鈕**
```
[執行下一次迭代]  [查看待審核知識]  [驗證效果]  [暫停]  [取消]
```

**API 需求**：
```
GET /api/v1/loops/{loop_id}
POST /api/v1/loops/{loop_id}/execute-iteration  （執行下一次迭代）
POST /api/v1/loops/{loop_id}/validate           （驗證效果）
POST /api/v1/loops/{loop_id}/pause              （暫停）
POST /api/v1/loops/{loop_id}/cancel             （取消）
```

|                                                                  |

## 7. 測試計劃

### 前端功能測試

**批量審核測試**（P0）：
1. 選取 10 個待審核知識
2. 點擊「批量批准」
3. 驗證：
   - [ ] 10 個知識狀態變為 'approved'
   - [ ] 10 個知識立即同步到 knowledge_base/vendor_sop_items
   - [ ] 前端顯示成功提示
   - [ ] 列表自動刷新

**啟動新迴圈測試**（P0）：
1. 點擊「啟動新迴圈」
2. 填寫表單（500 題，目標 85%）
3. 點擊「啟動」
4. 驗證：
   - [ ] 建立新的 loop 記錄
   - [ ] 自動執行第一次迭代
   - [ ] 前端跳轉到 loop 詳細頁面
   - [ ] 顯示回測進度

**執行下一次迭代測試**（P1）：
1. 審核完第一次迭代的知識
2. 點擊「執行下一次迭代」
3. 驗證：
   - [ ] 使用相同的 scenario_ids
   - [ ] 回測結果正確記錄
   - [ ] 改善幅度正確計算
   - [ ] 前端顯示迭代進度
