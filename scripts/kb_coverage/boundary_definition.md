# SOP 與 KB 分工邊界準則

## 定義

| 類型 | 全稱 | 用途 | 特徵 |
|------|------|------|------|
| **SOP** | Standard Operating Procedure | 操作性流程知識 | 包含具體步驟、觸發條件、可連結表單或 API 的多步驟流程 |
| **KB** | Knowledge Base | 解答性知識 | 直接回答問題、提供說明或指引，不涉及多步驟操作流程 |

## 判定決策樹

依照以下順序判定知識類型：

1. **涉及 2 步以上操作流程？** → **SOP**
   - 例如：報修申請流程、退租辦理步驟、繳費操作流程
2. **需觸發表單或 API？** → **SOP**
   - 例如：提交報修單、上傳合約、線上繳費
3. **回答「是什麼 / 為什麼」？** → **KB**
   - 例如：押金規定、管理費定義、違約金上限
4. **需查詢即時數據？** → **KB**
   - 例如：租金行情、帳單金額、繳費狀態

> **平手規則**：當 SOP 信號與 KB 信號數量相同時，預設歸入 KB。因為多數主題的本質是解答性知識，操作流程通常有明確的流程關鍵字。

## 同一主題並存原則

同一主題可同時擁有 SOP 和 KB 兩筆知識：

- **SOP** 負責「**怎麼做**」— 多步驟操作流程
- **KB** 負責「**是什麼 / 為什麼**」— 事實性解答

這兩者互補而非重疊。判斷時以**問題的意圖**區分：
- 使用者問「怎麼做」→ 導向 SOP
- 使用者問「是什麼 / 為什麼 / 多少」→ 導向 KB

## 對照範例

### 1. 押金（保證金）

| | SOP | KB |
|---|---|---|
| **主題** | 押金退還流程 | 押金規定說明 |
| **內容** | 租約到期 → 點交確認 → 扣除損壞費用 → 退還押金 → 簽收 | 法定上限為 2 個月租金，房東應於租約終止後返還，扣除條件包含修繕費用與欠繳租金 |
| **判定理由** | 多步驟操作流程，有觸發條件（租約到期），涉及簽收表單 | 解答「押金是什麼、上限多少、何時返還」，屬事實性知識 |

### 2. 報修

| | SOP | KB |
|---|---|---|
| **主題** | 報修申請流程 | 修繕責任歸屬說明 |
| **內容** | 租客提交報修單 → 管理師確認 → 派工 → 維修 → 驗收簽名 | 房東負責結構與主要設備維修，租客負責日常消耗品更換 |
| **判定理由** | 多步驟操作流程，需提交報修表單，涉及派工與驗收 | 解答「誰負責修繕」，屬責任歸屬的事實性知識 |

### 3. 管理費

| | SOP | KB |
|---|---|---|
| **主題** | 管理費繳費流程 | 管理費收費規則說明 |
| **內容** | 查詢帳單 → 選擇繳費方式 → 完成繳費 → 取得收據 | 包租業代收轉付模式與代管業服務費比例的差異解釋 |
| **判定理由** | 多步驟操作流程，涉及系統查詢與繳費表單 | 解答「管理費怎麼算、兩種模式差異是什麼」，屬規則說明 |

### 4. 合約（租賃契約）

| | SOP | KB |
|---|---|---|
| **主題** | 簽約流程 | 租賃契約必備條款說明 |
| **內容** | 審閱契約 → 確認條款 → 雙方簽署 → 上傳系統 → 存檔 | 定期 vs 不定期契約差異、契約審閱期規定、應記載事項 |
| **判定理由** | 多步驟操作流程，需簽署與上傳表單 | 解答「契約要包含什麼、有什麼類型」，屬法規與常識性知識 |

### 5. 繳費

| | SOP | KB |
|---|---|---|
| **主題** | 線上繳費操作流程 | 繳費方式與規定說明 |
| **內容** | 登入系統 → 查詢待繳帳單 → 選擇付款方式 → 確認付款 → 取得收據 | 可用銀行轉帳、超商代收、信用卡等方式，逾期需加收滯納金 |
| **判定理由** | 多步驟操作流程，需系統操作與付款表單 | 解答「有哪些繳費方式、逾期怎麼辦」，屬規定性知識 |

### 6. 退租

| | SOP | KB |
|---|---|---|
| **主題** | 退租辦理流程 | 提前解約條件說明 |
| **內容** | 提前通知 → 預約點交 → 現場點交 → 押金結算 → 歸還鑰匙 | 法定解約事由、違約金上限、通知期限規定 |
| **判定理由** | 多步驟操作流程，涉及通知、點交、結算等步驟 | 解答「什麼情況可以提前解約、違約金怎麼算」，屬法規知識 |

## 使用方式

### 程式化判定

```python
from scripts.kb_coverage.boundary_classifier import classify_knowledge_type

result = classify_knowledge_type("押金規定")  # → "kb"
result = classify_knowledge_type("報修申請流程")  # → "sop"
```

### 人工判定流程

1. 閱讀知識主題描述
2. 依照上方「判定決策樹」逐項檢查
3. 參考「對照範例」確認判定結果
4. 若同一主題需 SOP 和 KB 並存，分別建立兩筆知識
