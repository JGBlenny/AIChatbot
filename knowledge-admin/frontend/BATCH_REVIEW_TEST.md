# 批量審核功能測試指南

## 測試前準備

### 1. 重啟開發伺服器（清除緩存）

```bash
cd /Users/lenny/jgb/AIChatbot/knowledge-admin/frontend
npm run dev
```

### 2. 清除瀏覽器緩存

- **Chrome/Edge**: 按 `Ctrl+Shift+Delete` (Mac: `Cmd+Shift+Delete`)
- 選擇「快取的圖片和檔案」
- 點擊「清除資料」

或者使用無痕模式：
- **Chrome/Edge**: `Ctrl+Shift+N` (Mac: `Cmd+Shift+N`)

### 3. 訪問測試頁面

打開瀏覽器訪問：
```
http://localhost:8087/review-center
```

## 測試步驟

### 步驟 1: 切換到「迴圈知識審核」Tab

1. 在審核中心頁面，您應該看到 5 個 Tab：
   - 💡 意圖審核
   - 🧪 測試情境審核
   - ❓ 用戶問題
   - 📚 知識庫審核
   - **🔄 迴圈知識審核** ← 點擊這個

2. 點擊「🔄 迴圈知識審核」Tab

### 步驟 2: 確認批量審核 UI 元素

您應該看到以下新增的 UI 元素：

#### A. 全選區域（在篩選器下方）
```
┌─────────────────────────────────────────────┐
│ ☐ 全選（已選取 0 / X 項）                   │
└─────────────────────────────────────────────┘
```
- 藍色背景區塊
- 包含全選核取方塊

#### B. 每個知識卡片左側的核取方塊
```
┌───┬──────────────────────────────────────┐
│ ☐ │ #123 【SOP】租金繳納時間              │
│   │ 問題：租金每月幾號繳納？              │
│   │ ...                                  │
└───┴──────────────────────────────────────┘
```

#### C. 批量操作工具列（選取項目後顯示）
```
┌─────────────────────────────────────────────┐
│ ✓ 已選取 3 項  [清除選取]                   │
│                    [✅ 批量批准] [❌ 批量拒絕] │
└─────────────────────────────────────────────┘
```
- 橙色背景
- 顯示已選取數量
- 批量批准和批量拒絕按鈕

### 步驟 3: 測試全選功能

1. **測試全選**
   - 點擊全選核取方塊
   - ✅ 確認所有項目的核取方塊都被選中
   - ✅ 確認計數器顯示「已選取 X / X 項」

2. **測試取消全選**
   - 再次點擊全選核取方塊
   - ✅ 確認所有項目的核取方塊都被取消選中
   - ✅ 確認計數器顯示「已選取 0 / X 項」

3. **測試部分選取**
   - 手動勾選 2-3 個項目
   - ✅ 確認全選核取方塊顯示「部分選取」狀態（indeterminate）
   - ✅ 確認計數器顯示「已選取 2 / X 項」

### 步驟 4: 測試批量批准功能

1. **選取項目**
   - 勾選 3-5 個知識項目

2. **點擊「批量批准」按鈕**
   - ✅ 應顯示確認對話框

3. **確認對話框檢查**
   ```
   ┌─────────────────────────────────────────┐
   │ ✅ 批量批准確認                          │
   ├─────────────────────────────────────────┤
   │ 選取項目數量：5 項                       │
   │ SOP 項目：2 項                           │
   │ 一般知識：3 項                           │
   │                                         │
   │ 審核備註（選填）：                       │
   │ [_________________________________]      │
   │                                         │
   │          [取消]  [確定批准]              │
   └─────────────────────────────────────────┘
   ```
   - ✅ 統計資訊正確
   - ✅ 可輸入審核備註

4. **點擊「確定批准」**
   - ✅ 確認對話框關閉
   - ✅ 顯示進度對話框

5. **進度對話框檢查**
   ```
   ┌─────────────────────────────────────────┐
   │ ⏳ 批量審核進行中...                     │
   ├─────────────────────────────────────────┤
   │ 進度：3 / 5                              │
   │ 成功：3 項                               │
   │ 失敗：0 項                               │
   │                                         │
   │ [████████████░░░░░] 60%                 │
   │                                         │
   │ 正在處理：#456 - 租金繳納日期             │
   └─────────────────────────────────────────┘
   ```
   - ✅ 進度條動畫
   - ✅ 當前處理項目顯示
   - ✅ 統計即時更新

6. **結果對話框檢查**
   ```
   ┌─────────────────────────────────────────┐
   │ ✅ 批量審核完成                          │
   ├─────────────────────────────────────────┤
   │ ┌─────────┐  ┌─────────┐               │
   │ │ ✅      │  │ ❌      │               │
   │ │ 成功：  │  │ 失敗：  │               │
   │ │   5 項  │  │   0 項  │               │
   │ └─────────┘  └─────────┘               │
   │                                         │
   │              [確定]                     │
   └─────────────────────────────────────────┘
   ```
   - ✅ 成功/失敗統計卡片
   - ✅ 如果有失敗項目，顯示失敗清單

7. **點擊「確定」後**
   - ✅ 列表自動刷新
   - ✅ 統計卡片更新
   - ✅ 已審核的項目從列表中移除

### 步驟 5: 測試批量拒絕功能

1. **選取項目**
   - 勾選 2-3 個知識項目

2. **點擊「批量拒絕」按鈕**
   - ✅ 應顯示確認對話框

3. **確認對話框檢查**
   ```
   ┌─────────────────────────────────────────┐
   │ ❌ 批量拒絕確認                          │
   ├─────────────────────────────────────────┤
   │ 選取項目數量：3 項                       │
   │ SOP 項目：1 項                           │
   │ 一般知識：2 項                           │
   │                                         │
   │ ⚠️ 警告：拒絕的項目將無法復原，請確認您的決定。│
   │                                         │
   │ 審核備註（選填）：                       │
   │ [_________________________________]      │

   └─────────────────────────────────────────┘
   ```
   - ✅ 警告提示顯示（橙色背景）
   - ✅ 「確定拒絕」按鈕為紅色

4. 後續流程與批量批准相同

### 步驟 6: 測試重試失敗項目功能

此功能需要模擬失敗場景：

1. **創建失敗場景**
   - 選取包含 SOP 的項目
   - **不要**填寫業者或類別（這會導致驗證失敗）
   - 點擊「批量批准」

2. **觀察結果對話框**
   ```
   ┌─────────────────────────────────────────┐
   │ ⚠️ 批量審核完成（部分失敗）              │
   ├─────────────────────────────────────────┤
   │ ┌─────────┐  ┌─────────┐               │
   │ │ ✅      │  │ ❌      │               │
   │ │ 成功：  │  │ 失敗：  │               │
   │ │   2 項  │  │   1 項  │               │
   │ └─────────┘  └─────────┘               │
   │                                         │
   │ 失敗項目清單：                           │
   │ • #123 - 租金繳納規範                   │
   │   （SOP 項目缺少必填的業者或類別）       │
   │                                         │
   │          [確定]  [🔄 重試失敗項目]       │
   └─────────────────────────────────────────┘
   ```
   - ✅ 失敗項目清單顯示
   - ✅ 錯誤訊息顯示

3. **點擊「重試失敗項目」**
   - ✅ 自動選取失敗的項目
   - ✅ 重新顯示確認對話框
   - ✅ 可以修正問題後重新提交

## 常見問題排查

### 問題 1: 看不到批量審核 UI

**解決方案**：
```bash
# 1. 確認檔案已儲存
ls -lh /Users/lenny/jgb/AIChatbot/knowledge-admin/frontend/src/components/review/LoopKnowledgeReviewTab.vue

# 2. 確認檔案大小（應該約 35KB）
# 如果小於 20KB，代表修改沒有儲存成功

# 3. 重啟開發伺服器
cd /Users/lenny/jgb/AIChatbot/knowledge-admin/frontend
npm run dev

# 4. 強制重新載入瀏覽器
# Chrome: Ctrl+Shift+R (Mac: Cmd+Shift+R)
```

### 問題 2: 切換 Tab 後沒有內容

**解決方案**：
1. 打開瀏覽器開發者工具（F12）
2. 查看 Console 是否有錯誤訊息
3. 查看 Network 是否有 API 請求失敗

### 問題 3: 點擊批量按鈕沒有反應

**解決方案**：
1. 確認是否已選取項目（selectedIds.length > 0）
2. 打開 Console 查看是否有 JavaScript 錯誤
3. 檢查是否有 alert() 提示「請先選取要批准/拒絕的項目」

### 問題 4: 進度對話框沒有顯示

**可能原因**：
- API 請求過快完成
- showProgressDialog 狀態設置有問題

**檢查方式**：
1. 在 executeBatchReview 方法中加入 console.log
2. 觀察執行流程
